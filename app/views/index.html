Service Provider API Documentation

Authentication

POST /api/register

Register a new user

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123",
  "role": "client" // or "provider"
}

POST /api/login

Login a user

{
  "email": "john@example.com",
  "password": "password123"
}

Service Requests

GET /api/requests

Get all service requests

GET /api/requests/{id}

Get a specific service request

POST /api/requests

Create a new service request

{
  "title": "Fix leaking faucet",
  "description": "Kitchen faucet is leaking and needs repair",
  "location": "123 Main St",
  "user_id": 1
}

PUT /api/requests

Update a service request status

{
  "id": 1,
  "status": "accepted", // or "completed", "cancelled"
  "provider_id": 2 // optional, required for "accepted" status
}

Service Providers

GET /api/providers

Get all service providers

GET /api/providers/{id}

Get a specific service provider

POST /api/providers

Create a new service provider

{
  "user_id": 2,
  "name": "Jane's Repair Services",
  "services": "Plumbing, Electrical, Carpentry",
  "description": "Professional repair services with 10+ years of experience"
}

POST /api/providers/accept

Accept a service request

{
  "provider_id": 1,
  "request_id": 2
}

Users
